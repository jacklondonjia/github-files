Important information

  • The IAR C-SPY Angel driver can be used with evaluation boards that have the Angel ROM-monitor installed, or with JTAG interfaces that use the Angel protocol.

  • Both serial RS232 communication and Ethernet (TCP/IP) communication is supported.

  • See the QuickStart reference for Angel ROM-monitors and JTAG interfaces that helps you to set up some of the available evaluation boards.

Miscellaneous

  • Interrupts installed by the application, for example timer interrupts, will continue to run even when the debugger has stopped the target application. This may cause unexpected results. For example, setting a breakpoint in the timer interrupt handler after stopping the target application will crash the application. The main cause is that for communicating with the Angel debug monitor on the evaluation board, Angel uses the serial port and needs interrupts to be enabled, even when the target application has stopped.

  • There are some additional command line options for specific usage with the Angel driver, refer to the IAR Embedded Workbench Debugging Guide
